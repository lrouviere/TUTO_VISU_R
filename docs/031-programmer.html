4  Programmer

La programmation en R est basée sur les mêms principes que pour les autres logiciels de calcul scientifique. On y retrouve les tructures classiques de la programmation (boucles, condition if else…) et des fonctions prédéfinies pripres à la statistique classique.

4.1 Structures de contrôle

On énumère les principales structures :

  • Boucles for :

    for (i in vecteur){
  expr1
  expr2
  ...
}

    Par exemple

    for (i in 1:3){print(i)}
    [1] 1
[1] 2
[1] 3
    for (i in c("lundi","mardi","mercredi")){print(i)}
    [1] "lundi"
[1] "mardi"
[1] "mercredi"

  • Condition while

    while (condition) {expression}

    Par exemple :

    i <- 1
while (i<=3) {
  print(i)
  i <- i+1
}
    [1] 1
[1] 2
[1] 3

  • Condition if else

    if (condition){
  expr1
  ...
} else {
  expre2
  ...
}

    Par exemple :

    a <- -2
if (a>0){
  a <- a+1
} else {
  a <- a-1
}
print(a)
    [1] -3

  • switch

    switch(expression,
       "cond1" = action1,
       "cond2" = action2,
       ...)

    Par exemple :

    X <- matrix(0,nrow = 5,ncol = 5)
switch(class(X)[1],
       "matrix"=print("X est une matrice"),
       "data.frame"=print("X est un data.frame"),
       "numeric"=print("X est de classe numérique"))
    [1] "X est une matrice"

4.2 Écrire une fonction

On peut définir sa propre fonction dans un objet avec function :

mafonct <- function(param1,param2,...){
  expr1
  expr2
  return(...)
  }

Construisons par exemple la fonction factorielle qui calcule la factorielle d’un entier n :

factorielle <- function(n){
  return(prod(1:n))
  }

On peut la tester

factorielle(5)
[1] 120

On propose d’améliorer cette fonction en spécifiant des messages d’erreur ou d’alerte :

factorielle <- function(n){
  if (n<=0) stop("l'entier doit être strictement positif")
  if (ceiling(n)!=n) warning(paste("arrondi de",n,"en",ceiling(n)))
  return(prod(1:ceiling(n)))
  }

On a alors un message d’erreur lorsque le paramètre est négatif

factorielle(-2)
Error in factorielle(-2): l'entier doit être strictement positif

ou un avertissement si ce n’est pas un entier

factorielle(2.8)
Warning in factorielle(2.8): arrondi de 2.8 en 3
[1] 6

4.3 Les fonctions map

Ces fonctions appartiennent au package purrr du tidyverse. Elles permettent d’appliquer des fonctions à des listes et donc à des tibbles. On peut les voir comme des versions améliorées des fonctions apply. On peut par exemple retrouver les sorties de

apply(iris[,-5],2,mean)
Sepal.Length  Sepal.Width Petal.Length  Petal.Width 
    5.843333     3.057333     3.758000     1.199333

avec

map(iris[,-5],mean)
$Sepal.Length
[1] 5.843333

$Sepal.Width
[1] 3.057333

$Petal.Length
[1] 3.758

$Petal.Width
[1] 1.199333

ou encore

map_dbl(iris[,-5],mean)
Sepal.Length  Sepal.Width Petal.Length  Petal.Width 
    5.843333     3.057333     3.758000     1.199333

qui renvoie un vecteur de classe numeric plutôt qu’une liste. On peut citer également les fonctions map_lgl, map_chr, map_dbl qui retournent des vecteurs de logiques, de caractères ou d’entiers.

Dans le même style, on dispose des fonctions map2_... pour appliquer des fonctions à des paires d’éléments de listes. On peut par exemple sommer les élément de deux listes avec

l1 <- list(1,2,3)
l2 <- list(4,5,6)
map2(l1,l2,`+`)
[[1]]
[1] 5

[[2]]
[1] 7

[[3]]
[1] 9
#ou pour obtenir un vecteur
map2_dbl(l1,l2,`+`)
[1] 5 7 9

Il est également possible de spécifier explicitement sa propre fonction lorsqu’elle n’existe pas

set.seed(123)
tbl1 <- tibble(age=runif(5,20,50),taille=runif(5,150,180))
tbl2 <- tibble(age=runif(5,20,50),taille=runif(5,150,180))
map2_dbl(tbl1,tbl2,function(d1,d2) mean(rbind(d1,d2)))
      age    taille 
 38.21492 164.83358

La syntaxe peut paraître un peu lourde, avec notamment l’utilisation de function. On utilise régulièrement des fonctions anonymes qui peuvent se définir à l’aide d’une formule :

map2_dbl(tbl1,tbl2,~mean(rbind(.x,.y)))
      age    taille 
 38.21492 164.83358

ou en spécifiant explicitement les arguments

map2_dbl(tbl1,tbl2,\(d1,d2) mean(rbind(d1,d2)))
      age    taille 
 38.21492 164.83358

Notons enfin que l’utilisation des fonctions anonymes diffèrent lorsqu’on chaîne les opérations avec le pipe de la distribution de R de base |> ou avec celui de dplyr %>% :

set.seed(123)
X1 <- rnorm(100)
c(0.25,0.5,0.75) %>% quantile(X1,probs = .)
        25%         50%         75% 
-0.49385424  0.06175631  0.69181917 
c(0.25,0.5,0.75) |> quantile(X1,probs = .)
Error in if (na.rm) x <- x[!is.na(x)] else if (anyNA(x)) stop("missing values and NaN's not allowed if 'na.rm' is FALSE"): the condition has length > 1

Le . permet d’indiquer la place de la quantité à gauche du pipe %>%. Lorsqu’on utilise |>, il faut utiliser une fonction anonyme :

c(0.25,0.5,0.75) |> (\(p) quantile(X1,probs = p))()
        25%         50%         75% 
-0.49385424  0.06175631  0.69181917

    imputation <- function(tbl,choix="moyenne"){
  if (choix=="moyenne"){
    moy <- map_dbl(tbl,\(x) mean(x,na.rm = TRUE)) |> as.numeric()
    for (i in 1:ncol(tbl)){
      tbl[is.na(tbl[,i]),i] <- as.numeric(moy[i])
      }

        tbl[is.na(tbl[,i]),i] <- med[i]
      }
    }
  return(tbl)

Exercice 4.2 (Calculs d’indicateurs)  

  1. Écrire une fonction qui calcule la moyenne et la variance des colonnes d’un jeu de données qui ne contient que des variables continues. On utilisera une boucle for et on testera la fonction sur les 4 premières colonnes des données iris.

    moy_var <- function(tbl){
  p <- ncol(tbl)
  moyenne <- rep(0,p)
  variance <- rep(0,p)
  for (i in 1:ncol(tbl)){
    moyenne[i] <- mean(tbl[,i])
    variance[i] <- var(tbl[,i])
  }
  return(list(moy=moyenne,var=variance))
}
